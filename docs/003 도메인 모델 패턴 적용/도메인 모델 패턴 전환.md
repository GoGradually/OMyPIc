# 도메인 모델 패턴 전환 기록

## 목적

- 도메인 레이어를 단순 DTO 집합에서 벗어나, 애그리거트 단위의 모델/행위 중심 구조로 전환한다.
- 4개 레이어(`domain/application/presentation/infrastructure`)에서 같은 애그리거트 경계를 공유한다.

## 적용된 애그리거트

- `rulebook`
- `question`
- `session`
- `feedback`
- `wrongnote`
- `stt`
- `tts`
- 공통 요소는 각 레이어의 `shared` 패키지로 분리

## 전환 결과

- `domain`: 애그리거트별 엔티티/값 객체/도메인 로직 배치
- `application`: 애그리거트별 `usecase/port/model/policy` 배치
- `presentation`: 애그리거트별 `controller/dto` 배치
- `infrastructure`: 애그리거트별 어댑터(`persistence/gateway/storage/rag`) 배치
- `bootstrap`: 패키지 이동을 반영해 빈 조립 import 동기화

## 검증

- `./gradlew test` 기준 전체 백엔드 모듈 테스트 통과
